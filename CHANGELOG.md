# Changelog

## 0.6.1 — 2026-07-14

- 完成第二批互動接線：堂食、付款、重印及設定正式接入統一狀態模型。
- 堂食加入空枱開單、使用中枱詳情、追加單、2–10 份平均分單、支付一份、全數付款及混合付款入口。
- 付款彈窗支援現金、FPS、PayMe、Alipay、WeChat Pay 及混合付款，並建立 Payment Record。
- 堂食付款後即時更新已付／未付金額；購物車付款完成後清空目前訂單。
- 重印頁加入訂單選擇、收據／後廚／標籤目的地、多選及確認建立 Print Job。
- 更多／設定頁可即時切換分類每行、商品每行、購物車合併模式，並顯示自動完成、售罄及打印工作狀態。
- 新增 `src/interaction-v061.css`，補齊堂食分單、付款、重印及設定操作樣式。
- 本版本仍使用本地假資料，尚未接 Firebase、Worker、正式打印機或正式付款核對。

## 0.6.0 — 2026-07-14

- `App.tsx` 第一批正式接入 `useSmtState`／`smtReducer`，不再由多組分散的局部資料控制核心流程。
- 底部導航、分類、快找、購物車、件數、總額、角標全部改由統一狀態及 selector 計算。
- 商品加入、數量修改、商品備註、刪除、清空均改用 reducer action。
- 快速修改維持細型快捷卡；完整詳情仍為獨立大工作區。
- 掛單可建立正式本地快照，恢復後回到點單頁並還原購物車。
- 待補頁改用 Pending Issue 模型，支援逐項完成及按缺項批量完成。
- 工作台網絡單／異常單角標改由 Workbench Order 狀態即時計算；新單可接入進行中。
- 堂食使用中枱數、售罄數、系統異常數均改用統一 selector。
- 收款前會先攔截未完成待補；清空購物車加入掛單後清空及直接清空確認。
- 本版本仍使用本地假資料；付款、打印、堂食拆單及設定完整互動留待下一批。

## 0.5.0 — 2026-07-14

- 本階段先完成狀態模型骨架；下一階段把現有 `App.tsx` 所有局部 `useState` 逐步遷移到統一 reducer，並逐頁接通所有互動。
