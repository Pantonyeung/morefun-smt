# SMT UI-02A｜平板商品完整詳情工作區 V0.1 LOCK

日期：2026-07-13  
狀態：COMPLETE / LOCK  
設計基準：Sunmi T2s／Android 平板橫屏

## 1. 定位

UI-02A 不是獨立手機頁，而是嵌入 SMT 已鎖主介面內的商品詳情工作狀態。

- 保留 SMT 主狀態列。
- 保留主介面外框。
- 詳情內容在主工作區內切換。
- 完成後返回原分類、原商品位置及原購物籃位置。

## 2. 入口與完成

- 商品卡進入：完成後加入購物籃。
- 購物籃進入：完成後更新原商品。
- 組合入口進入：以原單點商品為基礎轉成自選飯團組合，不新增重複項目。
- 未按完成前只操作暫存資料。
- 返回即放棄本次修改，不彈確認視窗。
- 詳情頁／快捷卡的完成只代表保存本件商品設定。
- 真正必選檢查集中於整張訂單正式結帳前。

## 3. 候選池與快找

- 飯團、小食、飲品各自使用獨立候選池。
- 類內快找只縮小當前候選池，不可跨類擴大。
- 全局快找可以跨類別。
- 全局快找、快捷卡及詳情頁都固定有「售罄」入口及角標。
- 點「售罄」只顯示當前範圍內售罄商品。
- 永久停用商品不出現在點單介面。

## 4. 調整性選項

所有調整性選項統一：

- 預設收起。
- 需要時手動展開。
- 完成後自動關閉。
- 未調整時不顯示任何文案。
- 只顯示實際調整結果。

適用：

- 飯團配料。
- 沙律配料。
- 凍檸檬茶／水。
- 肉燥／咖喱／菜飯。
- 飯量、走蛋、醬汁、加料等。
- 快捷卡與完整詳情頁共用同一規則。

## 5. 自選飯團組合

單點飯團實際上可進入自選飯團工作區：

- 只有飯團：可完成。
- 飯團＋飲品：飲品用組合優惠價，可完成，建議加小食。
- 飯團＋小食：視為套餐方向，飲品在正式結帳前必選。
- 飯團＋小食＋飲品：完整自選飯團餐。

購物籃內單點飯團、小食、飲品均可使用「組合」按鍵導入自選飯團工作區。

## 6. 便當飯底

- 肉燥飯、咖喱飯、菜飯固定直接顯示。
- 三選一、必選、同價。
- 預設肉燥飯。
- 售罄者灰化、不可按，但不隱藏。
- 預設與 fallback 次序由 Admin／SMT 管理設定。
- 飯底決定正式產品身份：
  - 肉燥飯：12
  - 咖喱飯：C12
  - 菜飯：V12
- 顯示名稱、客人端、SMT、SMM、後廚單、標籤、收據、API、Firebase、Worker、Sheet V2 必須一致。

## 7. 紫米沙律

- 正常只有 S1–S7 七款固定紫米沙律。
- 正常時不顯示白飯／走飯。
- 只有全店紫米售罄時，先顯示白飯／走飯作替代。
- 醬汁第一項固定為「不需要」。
- 其餘四款醬由後台資料提供。
- 第二份醬不是第一層選項，而是獨立按鈕，固定 +$2。
- 選「不需要」時不可加第二份醬。

## 8. 備註

所有層級必須有備註：

- 商品詳情頁。
- 快捷卡。
- 單一商品。
- 自選組合。
- 整張訂單。
- 堂食、掛單、待補／重組、訂單修改。

商品備註與整單備註必須分層，並同步到後廚及打印輸出。

## 9. 價格

- 只顯示實際發生的加減價。
- 不顯示 +$0、未選、未套用等文案。
- 飯團＋飲品使用組合優惠價。
- 飯團＋小食＋飲品使用完整自選飯團餐價。
- 便當飯底同價，只改編號與名稱。
- 前端即時顯示；正式提交由 Worker 重新計價並保存快照。

## 10. 必選與異常

- 詳情頁／快捷卡不承擔過多阻擋。
- 可保存但未完成：待補。
- 原本有效但狀態改變：異常。
- 商品本身無法成立：不可成立。
- 正式結帳時統一檢查，並導入待補／重組區。

## 11. 合併、數量與打印

由後台設定：

- MERGE_IDENTICAL：完全相同商品可合併並顯示加減數量。
- SEPARATE_EACH：每件獨立列，不顯示加減。

只有商品、選項、調整、備註、價格版本及打印路由完全一致先可合併。

打印數量模式與購物籃顯示模式分離：

- 後廚單可展開多行。
- 標籤可打印多張。
- 收據可合併顯示數量。

## 12. 返回及狀態保存

必須保存：

- 入口來源。
- 商品及 cart item ID。
- 原分類。
- 商品列表及購物籃滾動位置。
- 快找條件。
- 暫存選項、調整、備註及價格。
