# SMT-UI-02B｜快速修改與快找邏輯鎖定

版本：V0.3
日期：2026-07-13
狀態：Idea Lock／Prototype Implemented

## 1. 快速修改

購物車單項修改只保留一個主操作：`完成`。

- 沒有修改，按完成：直接關閉。
- 已有修改，按完成：保留修改並關閉。
- 不提供重選、儲存、套用等重複主操作。

摘要項目：

- 飯團
- 小食
- 飲品
- 數量
- 備註
- 查看完整詳情
- 完成

## 2. 取消文字搜尋

SMT 不以鍵盤文字搜尋作主要找商品方式。V0.1 Prototype 不提供文字搜尋框。

## 3. 兩套快找必須隔離

### 3.1 全店快找

- 使用範圍：開單產品區。
- 可以跨商品類別。
- 標籤由 Admin 控制。
- 結果必須仍受目前畫面分類條件約束；日後如改成真正跨類結果，必須按商品類型分組顯示。

### 3.2 類內快找

- 使用範圍：飯團、小食、飲品等選擇頁。
- 只可縮小目前候選池。
- 不可擴大候選池。
- 不可顯示其他商品類型。
- 不可跳離目前選擇流程。

例：

```text
候選池 = 飯團
快找標籤 = 雞
結果 = 具有「雞」標籤的飯團
```

禁止出現雞扒便當、炸雞小食或其他非飯團商品。

## 4. 無結果行為

類內快找無結果時：

- 顯示目前類別沒有該標籤選項。
- 提供「查看全部此類」操作。
- 不可自動跨類找結果。

## 5. Admin 控制項目

後續 Admin 應可控制：

- 快找標籤名稱
- 排列次序
- 顯示／隱藏
- 標籤適用畫面
- 商品與標籤關係
- 商品候選角色，例如 `rice_ball_choice`、`snack_choice`、`drink_choice`

## 6. Prototype 實作狀態

目前 `src/App.tsx` 已實作：

- 開單產品卡
- 購物車
- 快速修改抽屜
- 飯團／小食／飲品類內選擇
- 類內快找候選池隔離
- 無結果空狀態
- 單一「完成」操作

目前全部使用本地假資料，未連接正式 Firebase、Worker、Staff／Sync API 或打印服務。
