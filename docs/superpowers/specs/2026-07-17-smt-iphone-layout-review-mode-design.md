# SMT iPhone 橫屏 Layout Review Mode 設計

## 目的

在 SMT 正式 T2s／大平板版面未完全封板前，提供一個可於 iPhone 橫屏直接檢查 UI 排版、元件位置、資訊層級及互動流程的臨時驗收版本。

此版本不是最終營運尺寸，也不是獨立手機版。它只作開發及封板期間的快速視覺檢查工具。

## 核心原則

1. T2s、iPad 橫屏、iPhone 橫屏共用同一頁面結構。
2. 元件順序、功能位置、導航位置及操作流程必須一致。
3. iPhone 只可縮小字體、圖片、間距、內距及欄高。
4. 不可因 iPhone 而重排成上下結構、漢堡選單、抽屜式主導航或精簡功能版。
5. 所有 Layout 在 iPhone 確認後，再按相同比例擴展成 T2s 正式尺寸。

## 固定結構

- 頂部全局狀態列
- 中央工作區
- 底部五項導航：點單｜訂單｜堂食｜售罄｜更多
- 點單頁固定左 30% 訂單工作區、右 70% 商品工作區
- 堂食固定左側枱單／輪候摘要、右側 3×3 九宮格及候位工作區
- 商品詳情使用右側操作面板，左側訂單保持可見

## iPhone 橫屏密度規則

- 使用 `100dvh`／`100svh`，不得使用固定最小高度
- 頂欄約 42–46px
- 底欄約 48–52px
- 左右 Safe Area 必須套用
- 商品每行最多 4 格
- 分類仍維持 2×7
- 快捷飲品維持一行，不得隱藏
- 文字可縮小但不得截斷核心狀態
- 圖片高度可縮小
- 中央各區域各自內部捲動
- 快速模式、狀態入口及五項導航不可隱藏

## 允許差異

- 字體尺寸
- 圖片高度
- 卡片內距
- 元件間距
- 頂欄與底欄高度
- 商品 Grid 由 5 格縮至 4 格

## 禁止差異

- 更改導航位置
- 更改功能順序
- 隱藏快捷飲品
- 隱藏快速模式
- 改變堂食九宮格
- 改變點單 30／70 結構
- 將商品詳情改為全屏獨立頁
- 將 iPhone 變成另一套手機版流程

## 驗收裝置

1. iPhone 橫屏：主要開發期 Layout 驗收
2. iPad 橫屏：中尺寸驗收
3. Sunmi T2s／Android Kiosk：最終正式尺寸與實機驗收

## 成功標準

- iPhone 橫屏可完整看到頂欄、主工作區及底部導航
- 所有核心操作位置與 T2s 版本一致
- 只出現密度差異，不出現流程或結構差異
- 版面封板後，可直接放大成 T2s 正式尺寸而不需要重新設計
